[케이스 ID] BTC-006
[발견일] 2026-05-13
[제목] Vercel 배포 환경에서 로봇/담당자 사진이 전혀 안 뜸

[행동]
Vercel에 배포된 주소로 접속했어요. 로그인하고 전체 현황 화면을 봤는데
로봇 카드에 사진이 있어야 할 자리가 비어 있어요.
담당자 목록에서도 사진이 있는 분들 자리가 다 비어 있어요.

[결과]
- 로봇 카드: 이미지 자리가 비어 있음 (placeholder 또는 아이콘으로 대체)
- 담당자 카드: 사진 자리 비어 있음
- 로컬(Docker)에서는 동일한 사진이 정상 표시됨

[기대]
로봇 사진과 담당자 증명사진이 보여야 해요.

[재현 가능] 항상 (Vercel 환경에서)

[심각도 (사용자 관점)] 불편하다

---

**기술적 원인 (QA 분석용):**
Vercel이 `outputDirectory`를 명시하지 않으면 `public/` 폴더를 웹 루트로 사용함.
이미지 경로가 `../src/assets/images/...` → 웹에서 `/src/assets/images/...`로 해석되지만
Vercel 루트가 `public/`이므로 `public/src/assets/images/...`를 찾아 404 발생.

**수정 방법:**
`vercel.json`에 `"outputDirectory": "."` 추가 → Vercel이 프로젝트 루트 전체를 서빙.
